## 填写指南

### 各章节填写说明

| 章节 | 填写说明 | 注意事项 |
|------|----------|----------|
| 1. 需求概述 | 简洁描述需求核心，便于快速理解 | 避免技术术语，用业务语言 |
| 2. 动机分析 | 深入挖掘"为什么要做"，明确价值 | 价值尽量量化，便于后续评估 |
| 3. 范围定义 | 明确做什么、不做什么，防止范围蔓延 | MVP 聚焦核心功能，二期功能明确标注 |
| 4. 用户流程 | 描述用户视角的操作流程 | 可配合流程图，更直观 |
| 5. 数据需求 | 明确数据来源和字段 | 考虑数据权限和隐私 |
| 6. 非功能需求 | 性能、安全、兼容性等要求 | 要求要具体可衡量 |
| 7. 验收标准 | 明确的完成标准 | 便于 QA 编写测试用例 |
| 8. 风险与依赖 | 提前识别风险，确认依赖 | 风险要有缓解措施 |

### 常见错误

❌ **错误示例**：
- 需求描述模糊："提升用户体验"
- 范围不清晰："等后续再确定"
- 验收标准不可衡量："性能要好"
- 风险未识别："应该没什么风险"

✅ **正确示例**：
- 需求描述具体："将页面加载时间从 3 秒降至 1 秒"
- 范围明确："一期支持 3 种筛选条件，二期扩展至 10 种"
- 验收标准可衡量："95% 的请求响应时间 < 1 秒"
- 风险有应对："依赖方可能延期，已确认备选方案"
