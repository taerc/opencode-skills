## 填写指南

### 各章节填写说明

| 章节 | 填写说明 | 注意事项 |
|------|----------|----------|
| 1. 方案概述 | 清晰描述方案背景和目标 | 让非技术人员也能理解 |
| 2. 架构分析 | 详细描述技术方案 | 配图更直观，关键决策要说明理由 |
| 3. 可行性评估 | 客观评估技术、资源、时间可行性 | 不回避问题，如实评估风险 |
| 4. 风险识别 | 全面识别技术和管理风险 | 每个风险都要有缓解措施 |
| 5. 依赖分析 | 明确内外部依赖 | 提前确认依赖方计划 |
| 6. 性能评估 | 量化性能目标和评估 | 压测计划要具体可执行 |
| 7. 安全评估 | 评估安全风险和合规 | 安全问题不能妥协 |
| 8. 可维护性 | 评估运维和扩展能力 | 考虑长期运维成本 |
| 9. 建议 | 分优先级给出改进建议 | 明确哪些是必须改进的 |

### 常见错误

❌ **错误示例**：
- 架构图缺失或不清晰
- 技术选型没有备选方案对比
- 风险评估流于形式，没有缓解措施
- 性能目标不量化
- 依赖项未识别或状态不明确

✅ **正确示例**：
- 架构图清晰展示组件关系和数据流
- 技术选型有明确的选择理由和备选方案
- 风险有等级、有影响、有缓解措施
- 性能目标量化（QPS、响应时间、可用性）
- 依赖项有负责方、交付时间、状态跟踪
