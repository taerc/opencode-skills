## 填写指南

### 各章节填写说明

| 章节 | 填写说明 | 注意事项 |
|------|----------|----------|
| 1. 项目概述 | 清晰描述项目背景和目标 | 让决策者快速理解项目价值 |
| 2. 需求分析 | 评估需求的确定性和稳定性 | 需求不确定性是项目失败的主因之一 |
| 3. 技术可行性 | 客观评估技术能力和风险 | 不回避技术难点，如实评估 |
| 4. 资源可行性 | 详细列出人力、预算需求 | 预算要合理，留有余地 |
| 5. 时间可行性 | 评估时间是否充足 | 考虑依赖和风险对时间的影响 |
| 6. 商业可行性 | 量化商业价值和 ROI | 商业价值要可衡量 |
| 7. 风险评估 | 全面识别各类风险 | 每个风险都要有应对方案 |
| 8. 综合评估 | 量化评分支持决策 | 评分要客观，有依据 |

### 常见错误

❌ **错误示例**：
- 需求描述模糊，需求确定性未评估
- 技术风险轻描淡写，没有缓解措施
- 预算估算过于乐观，没有 contingency
- 时间规划不考虑依赖和风险
- 商业价值无法量化

✅ **正确示例**：
- 需求有明确来源和验证
- 技术风险有等级、有应对方案
- 预算有明细，包含 10-20% contingency
- 时间规划考虑关键路径和风险缓冲
- 商业价值有量化指标和计算依据
