# AI 能力边界清单

## LLM 适用场景评估

| 场景类型 | 适合度 | 说明 |
|----------|--------|------|
| 内容生成 | ✅ 高度适合 | 文案、代码、图像、视频生成 |
| 文本分类 | ✅ 高度适合 | 情感分析、意图识别、标签分类 |
| 信息提取 | ✅ 高度适合 | 实体抽取、关键信息提取 |
| 对话交互 | ✅ 高度适合 | 客服、助手、问答系统 |
| 数值计算 | ⚠️ 谨慎使用 | LLM 不擅长精确计算，需外部工具 |
| 实时数据 | ⚠️ 谨慎使用 | 需配合外部数据源和 RAG |
| 精确逻辑 | ⚠️ 谨慎使用 | 复杂推理可能出错，需验证 |
| 高可靠系统 | ❌ 不适合 | 金融交易、医疗诊断等需传统系统 |
| 敏感数据处理 | ❌ 不适合 | 需考虑数据隐私和合规要求 |

## AI 技术选型指南

### RAG vs Fine-tuning 选型

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 知识频繁更新 | RAG | 无需重新训练，实时更新 |
| 需要专业知识 | RAG | 可检索外部知识库 |
| 特定风格输出 | Fine-tuning | 模型学习特定风格 |
| 领域专业术语 | Fine-tuning | 模型内化领域知识 |
| 成本敏感 | RAG | 无训练成本 |

### Agent 系统设计要点

1. 任务拆解能力：Agent 需要将复杂任务分解为可执行步骤
2. 工具调用可靠性：确保工具调用的正确性和容错
3. 人机协作边界：明确哪些决策需要人工确认
4. 错误处理和回退：设计合理的失败处理机制
